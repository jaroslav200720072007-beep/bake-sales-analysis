Проєкт присвячений глибокому аналізу діяльності американської мережі магазинів велосипедів. Я дослідив дані за 2016–2018 роки, щоб перетворити суху статистику на конкретні бізнес-рішення.

## 🔍 4. Глибоке занурення у висновки (Deep Dive)

У цьому розділі я розкриваю логіку аналізу: від виявлення бізнес-проблеми до її технічного підтвердження за допомогою SQL та фінальної візуалізації в Tableau.

### 📈 А. Сезонність та пікові навантаження
**Проблема:** Необхідність точного прогнозування складських запасів для запобігання дефіциту під час сплесків активності.
**Аналіз:** Вивчення часових рядів за 2016–2018 роки показало, що березень та квітень є критично важливими місяцями. Особливу увагу привертає квітень 2018 року — зафіксовано аномальний стрибок продажів перед стабілізацією.

*(На графіку **Revenue / Month**)*.

*   **Технічне підтвердження:** Для отримання цих цифр я використав SQL-агрегацію (номер 12 у скрипті[SQL (PostgreSQL/MySQL)](./bikesales-script.sql)):
```sql
-- Аналіз доходів на місяць для виявлення сезонності
SELECT 
    YEAR(order_date) AS year,
    MONTH(order_date) AS month,
    SUM(revenue) AS total_revenue
FROM "BikeStores.xlsx - Data"
GROUP BY year, month
ORDER BY year, month;
```
### 🚲 Б. Продуктова стратегія та лідери ринку

*   **Технічне підтвердження:** Для виділення ТОП-3 товарів у кожній категорії я застосував віконну функцію `RANK()` (номер 13 у SQL-скрипті[SQL (PostgreSQL/MySQL)](./bikesales-script.sql)):
```sql
-- Ранжування товарів усередині кожної категорії за кількістю продажів
WITH top_3 AS (
    SELECT category_name, product_name, total_units,
    RANK() OVER (PARTITION BY category_name ORDER BY total_units DESC) AS rnk
    FROM "BikeStores.xlsx - Data"
)
SELECT category_name, product_name, total_units
FROM top_3 
WHERE rnk <= 3;
```
### 👥 В. Утримання клієнтів (Customer Retention)
**Проблема:** Висока вартість залучення нових клієнтів при низькій лояльності (більшість покупців не повертаються за повторною покупкою).
**Аналіз:** Розрахунок показника Retention показав, що понад **85% клієнтів** здійснюють лише одну покупку[span_0](start_span)[span_0](end_span). Це свідчить про те, що бізнес має величезний потенціал для зростання через впровадження маркетингу та програм лояльності для стимулювання повторних замовлень.

*   **Технічне підтвердження:** Я розрахував відсоток клієнтів, що повернулися, за допомогою CTE та агрегатних функцій (номер 15 у SQL-скрипті [SQL (PostgreSQL/MySQL)](./bikesales-script.sql)):
```sql
-- Розрахунок відсотка Retention (клієнти, що зробили більше 1 замовлення)
WITH customer_orders AS (
    SELECT customers, COUNT(order_id) AS total_orders
    FROM "BikeStores.xlsx - Data"
    GROUP BY customers
)
SELECT 
    ROUND(SUM(CASE WHEN total_orders > 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS returning_percentage
FROM customer_orders;
```
